# Security Policy

We take the security of flarecrawl seriously. Thank you for taking the time to
responsibly disclose any issues you find.

## Supported Versions

Security fixes are applied to the latest release. If you are running an older
version, please upgrade before reporting.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests** — public disclosure before a fix is available
puts every user at risk.

Report privately using **GitHub's private vulnerability reporting**:

1. Go to the **Security** tab of this repository.
2. Click **Report a vulnerability** (under *Advisories*).
3. Include a description and impact, steps to reproduce (proof-of-concept,
   affected version, configuration), and any known mitigations.

## What to Expect

| Stage               | Target                                              |
| ------------------- | --------------------------------------------------- |
| Acknowledgement     | within **3 business days**                          |
| Initial assessment  | within **7 business days**                          |
| Status updates      | at least every **14 days** until resolved           |
| Public disclosure   | coordinated with you, typically after a fix ships   |

We will credit you in the advisory unless you ask to remain anonymous.

## Scope

In scope: the code in this repository and its official release artefacts, on
supported versions. Out of scope: third-party dependency vulnerabilities (report
upstream; a heads-up is appreciated), issues requiring physical access or social
engineering, and unrealistic resource-exhaustion DoS.
